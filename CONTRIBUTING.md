# Contributing to Awesome Tech Interview Prep

Thank you for your interest in contributing! This list aims to be a genuinely useful resource for anyone preparing for tech interviews.

## Guidelines

### What to add

- **Free resources** are strongly preferred (open-source projects, free courses, blog posts, guides)
- **Paid resources** (books, courses) are welcome if they are widely recognized and genuinely helpful
- Resources should be **actively maintained** (updated within the last 12 months)
- Each resource should include a brief, accurate description

### What NOT to add

- Self-promotional links without genuine value
- Affiliate links (use clean URLs)
- Resources behind aggressive paywalls with no free content
- Duplicate resources that cover the same ground as existing entries

### Format

Each entry should follow this format:

```markdown
- [Resource Name](https://url.com) - Brief description of what it offers and why it is useful.
```

For books, include the author:

```markdown
- [Book Title](https://url.com) by Author Name - Brief description.
```

### Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/add-resource`)
3. Add your resource in the appropriate category
4. Ensure your addition is in alphabetical order within its subsection
5. Commit with a clear message (`git commit -m 'Add [Resource Name] to [Category]'`)
6. Push and open a Pull Request

### Categories

If your resource does not fit an existing category, propose a new one in your PR description with justification.

## Code of Conduct

Be respectful and constructive. This is a community resource meant to help people succeed in their career journeys.
